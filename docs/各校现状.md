## 通用机制备忘

- **数据体检**：`./run.sh audit`——13 断言/校，红灯退出码 1，挂在
  `./run.sh` 日常流程末尾；每校差异走 yaml `expect:` 段（阈值/跳过+原因）。
- **验收三步法**：完整性 diff（防爬漏）→ 课程名单三问（防源头缺登记）→
  覆盖率+分布（防解析浅）。见 [数据策略.md](数据策略.md) 第〇节。
- **抓取方式**：html / portlet_post（uPortal 预热+POST）/ 域级 prime 链
  （会话态 servlet，yaml domains 配置）/ js_render+pdf（v1 不抓，预留）。
- **派生回填**：`crawler/derive.py`（run.sh 自动调）——源站没有但能从
  已有数据推出的字段，幂等只填空。规则集中在此，**禁止散落一次性 SQL**。
  现有：专业归属 ← 课表课程多数票。
- **归属通道**：结构化源优先（meta/JSON 字段/URL 参数/目录分区）；无归属
  字段的走「课表引用」（前端显形）或「课表多数票」（derive 反推）。措辞
  变体进 yaml `faculty_alias`，无信号进 `faculty_overrides` 人工层。
- **限流经验**：华威主站、KCL Akamai 均为报复性限流——高并发只给实测
  无限流的域（利兹、courses.warwick、澳洲各 CDN）；新域一律保守起步。
  **Playwright/CDP 对限流无用**（真浏览器一样被数）。
- **CDP 抓取方式**（`fetch_method: cdp`，browser.py）：应对**挑战型 WAF**，非限流。
  有头 Chrome 持久 profile，用户手动过一次验证后跨运行复用。两套抓法按 WAF 类型
  分：①**token/cookie 型**（UQ AWS WAF）走 `ctx.request`（快），解完保留守护页自动
  续 token；②**行为型**（墨尔本 Incapsula）走 `page.goto` 真导航（`navmode` 自动切）。
  拦截页伪装成 200 空壳的（Incapsula）必须列入 `CF_MARKERS` 防污染。Incapsula 还会
  IP 硬封需冷却，不适合大站快抓。
- **抖动**：`min(JITTER, interval)`——高速档（0.5s/0.2s）曾被固定 0~1s
  抖动双倍拖慢，已改上限跟随间隔。
- **速度两极同一套码**：阿德莱德 0.2s×6 ≈200 页/分钟；华威主站 429 泥潭
  ≈1.5 页/分钟。按域实测定档，别猜。
