## 墨尔本 melb（2026-07-05 调研）

**死胡同（勿重试）**
- 主站三个域 `www / students / study.unimelb.edu.au` 全部 Akamai 403 硬挡：
  加全套浏览器头无效，WebFetch 也 403。官方 principal dates 页
  （`unimelb.edu.au/dates`）因此脚本不可达。
- `handbook.unimelb.edu.au/2026/courses?page=N` 形式分页**无效**（返回空列表）——
  真实分页是 `/search?page=N&types[]=course|subject&year=YYYY`（从导航 href 取）。
- handbook 域连发 ~6 请求触发 Imperva「Pardon Our Interruption」；5s 间隔 +
  单并发未触发。封锁分两级：先按**客户端指纹**（curl 被封时 aiohttp 一度照常通过），
  继续触发后升级为 **IP 级**（aiohttp 也被挑战），且有粘性（静默 20 分钟不解）。
  教训：手工诊断的每次试探都在喂它——确认「能抓」后就停手，交给管道跑；
  被封时按 CF_BACKOFFS 退避等下轮（管道已自动如此），恢复检测用 ≥45 分钟
  间隔的单发 aiohttp。
- `handbook.unimelb.edu.au/2026/courses|subjects` 会 **301 到
  `/search?year=YYYY&types[]=course|subject`**——管道自动登记跳转目标为新任务，
  解析器对两种 URL 通用；勿把 301 当页面失效。
- `students.unimelb.edu.au` 的 2026 Wall Planner PDF 同样 403，不可作校历源。

**人工数据来源**

  source_page=15570）录自 **Universities Australia《Academic calendar 2026-27》PDF

## 悉尼 sydney（2026-07-05，踩坑最多）

**数据源分层**
- 主站 `www.sydney.edu.au` **无 WAF**，全可抓。官方目录（handbook/units）均 JS 壳，
  数据在 **Coveo 搜索**后面。org+apiKey 从 clientlib `corporate-search.*.js` 提出
  （`universityofsydneyproduction10somjans` / `xxf8204f8d-...`，公开客户端令牌），
  Coveo REST **支持 GET**（access_token 入 query）→ 查询 URL 当种子页解析 JSON。
- 专业：Coveo `Courses` 源（营销层，clickUri 是招生页），字段够全直接建库
  （studylevel→级别、fieldofstudy→学院，2026 过滤）。归属 92%，缺的 8% 按学位名
  关键词补到 99%（学位名自带学科，非正文启发式）。
- 课程：Coveo `Units of Study [Sitemap]` 源枚举 + `/units/{code}` SSR 详情页取名称/学分。

**死胡同 / 坑（血泪）**
- **CUSP**（cusp.sydney.edu.au）是行政层但**只覆盖 3 个院系**（工程/健康/建筑），
  其余 6 院未迁入，当不了主源。它是 jQuery+jsTree，树展开协议已逆向
  （`/students/view-degree-programs-tree-node-ajax/did/{院系}/.../node_pid/{数字id}`，
  递归拼 id），但覆盖不全故弃用。
- 传统 handbook（`/handbooks/{院系}/`）：landing 页是 AEM 空壳；真实内容在深层
  `.../overview.html|course-resolutions.html|unit-of-study-table.html`（Coveo
  `Handbooks Sitemap` 源，9006 条，URL 段带院系=行政层），但命名各院不统一、
  混学位与 major、Coveo 有 5000 条上限——提纯成本高，未采用。
- **2026 的 session URL 大量未上线**：`/units/{code}/{2026-S1C-ND-CC}` 多数 302 到
  `/errors/500.html`（丢 74%）。改用**裸 `/units/{code}`**（直接 200，天然按代码去重）。
- **裸页学分正则**：`page.re` 搜去标签纯文本，故用 `Credit points\s+(\d+)`（标注式）；
  松散的 `(\d+) credit point` 会抓到学位总学分（如 144），勿用。
- **瞬时 500 被误判 moved**：并发下部分裸 URL 302→/errors/500，pipeline 判永久
  moved 不重试。重置 status=active 重抓即恢复。（pipeline 缺陷：重定向目标是
  /errors/50x 时应按瞬时失败而非 moved——待修。）
- **裸 URL 页无学院信号**（session 页正文有「Faculty of X」，裸页没有）。课程归属
  靠**学科码前缀→学院**：老 session 快照自动提 82 前缀 + 手工标准学科码表，覆盖 80%。
  这是前缀推断（CIVL→工程），非页面权威信号——与墨/莫（JSON 直给学院）质量不同。
