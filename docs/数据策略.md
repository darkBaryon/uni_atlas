# uni_atlas 取数据策略（2026-07-04 定）

> 业务定位：**留学辅导机构**（帮在读学生过课），不是申请机构。
> 核心数据四件套：**校历考试期 / 院系结构 / 课程名单+官方链接 / module_contents 经验层**。
> 克制原则：官方维护完善的资源存链接不镜像；机构自有经验才是要沉淀的资产。

## 二、澳洲/香港怎么做

**结构**（全部现成）：`config/universities/au|hk/` + `parsers/au|hk/{common,generic}.py`；
国家层 `_defaults.yaml` 定货币（AUD/HKD）与学年月份。

**每校流程**（辅导镜头，估 1-2 小时/校）：
1. 调研 4 个入口：校历页、院系列表、官方课程目录（存链接）、专业目录（课程发现用）
2. 写 yaml（**不 seed 招生/学费/语言页**——从第一天就不采申请域）
3. 解析器只写两个：`term_dates`（优先，辅导核心）+ `program_catalog`；
   能用 GenericXX + yaml 就不写 Python
4. `./run.sh check <校>` 覆盖率验收

**校单**：取自《中国留学生热门大学名单》——澳八大 + 港校。

**已知改造点**：澳洲学年 2 月开学（S1 二至六月、S2 七至十一月），
前端「当前学年」过滤现在写死英式 9 月起算——地区扩展前须按 country 分支。

## 三、前端：按地区分组

建议：**总览页按地区分节**（英国 / 澳洲 / 港澳三个分组标题），**不加路由层级**。
- 依据：学校总数 <30 时，分组标题的扫视成本低于多一次点击；
  universities.country 字段现成，零 schema 改动
- 升级信号：单个地区超过 ~15 校，或加入美加等更多地区时，再考虑地区页/折叠
